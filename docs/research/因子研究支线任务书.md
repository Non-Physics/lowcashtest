# 因子研究支线任务书

## 1. 任务定位

这是一条独立的“因子选股研究支线”。

它的目标不是立刻改造当前交易系统，也不是直接产出可下单的新策略，而是建立一套可复现、可比较、可逐步接入主线的因子研究框架。

这条支线必须服务于当前主线，而不是打断当前主线。

## 2. 当前主线背景

当前项目已经具备以下基础：

- 本地股票数据链路已打通
- 基线策略可复现
- 信号语义已固定：
  - 收盘后生成信号
  - 下一交易日开盘执行
  - 先卖后买
- 模拟盘半自动交易闭环已基本打通
- 券商读取主路径目前收口为 `PDF -> OCR`

当前主线的工作重点是：

- 继续优化现有基线策略
- 维持交易系统稳定

因此，因子研究是独立支线，不应影响当前交易主线。

## 3. 这条支线要回答的问题

这条支线最终要回答下面几个问题：

1. 因子是否对股票横截面收益有区分能力
2. 因子组合是否比当前基线选股更有增益
3. 因子效果是否能经受时间切分验证，而不是只在全样本有效
4. 将来如果要接入主线，最自然的接入点在哪里

## 4. 必须遵守的边界

### 4.1 不要碰的地方

- 不要修改当前默认交易入口
- 不要修改同花顺交易适配和半自动流程语义
- 不要把因子实验直接接入当前交易执行链
- 不要污染当前主线输出目录
- 不要把研究脚本和生产脚本混在一起
- 不要为了做因子研究而大规模重构整个项目
- 不要一开始就堆太多因子或做复杂模型

### 4.2 必须保留的前提

- 必须保留当前基线策略作为对照组
- 所有研究结论都要回答：
  相比当前基线，因子是否真的提供了增益
- 必须做时间切分验证
- 必须保证标签定义与当前主线时序兼容

## 6. 先做什么，不先做什么

### 6.1 第一阶段只做研究框架

第一阶段只做研究框架，不做交易接入。

优先级如下：

1. 盘点当前项目中可复用的内容
2. 设计独立的因子研究目录
3. 实现最小因子计算与评估流程
4. 产出研究输出和总结文档
5. 给出未来接入主线的方案，但先不要接

### 6.2 暂时不要做的事

- 不要改 `股票策略交易主流程.py` 的主交易语义
- 不要改 `模拟盘半自动日常.py` 的日常流程
- 不要重新引入复杂的交易端依赖
- 不要把因子研究结果直接写进现有交易状态
- 不要为了看起来高级就上来做模型集成或深度学习
- 不要跳过时间切分验证
- 不要只看收益，不看回撤和稳定性

## 7. 优先复用的现有资产

支线开始前，请优先梳理并复用这些能力：

### 7.1 数据层

- 本地 30 分钟数据
- 交易日历

### 7.2 策略层

- 当前基线策略脚本
- 当前信号生成服务
- 当前时间切分 / 参数优化思路

### 7.3 输出层

- 当前 `outputs/` 的组织方式
- 当前研究类输出的命名方式

### 7.4 文档层

- [README.md](/mnt/d/量化/同花顺/pythonProject/README.md)
- [docs/guide/项目导览.md](/mnt/d/量化/同花顺/pythonProject/docs/guide/项目导览.md)

目标不是重写这些东西，而是尽量借力。

## 8. 因子第一版的范围建议

第一版不要做复杂机器学习，不要做黑箱打分。

先做 3 到 5 个解释性强的因子，例如：

### 8.1 动量类

- 20 日收益
- 60 日收益

### 8.2 趋势类

- 收盘价相对 MA20 / MA60 偏离
- MA20 斜率

### 8.3 风险类

- ATR / 波动率
- 近期最大回撤

### 8.4 成交类

- 成交额均值
- 量价配合强度

如果项目已有指标可直接复用，优先复用。

## 9. 研究框架必须回答的核心问题

每个因子都至少要回答：

1. 因子值如何定义
2. 因子在哪一天可见
3. 因子是否存在未来函数风险
4. 因子对下一交易日或未来 N 日收益是否有区分度
5. 因子效果在不同时间段是否稳定
6. 因子与当前基线选股逻辑是互补还是冲突

## 10. 研究流程建议

建议按下面顺序推进：

1. 定义股票池
2. 计算候选因子
3. 生成因子面板
4. 计算未来收益标签
5. 做分层检验 / 分组收益
6. 做时间切分验证
7. 做简单因子组合
8. 与当前基线结果对比
9. 输出报告和结论

## 11. 股票池要求

股票池必须明确，不允许模糊。

至少要说明：

- 是否过滤 ST
- 是否过滤停牌
- 是否过滤上市未满 N 天
- 是否设置流动性门槛
- 是否限制涨跌停不可买情形
- 是否与当前基线股票池一致

优先建议先与当前基线股票池保持一致，这样比较更干净。

## 12. 标签定义要求

标签必须与当前主线时序兼容。

第一版建议固定一种简单标签，例如：

- 下一交易日开盘到未来第 N 日收盘收益

或者：

- 下一交易日开盘到下一交易日收盘收益

不要在第一版里混用太多标签，否则解释会很乱。

## 13. 评估要求

至少提供这些评估：

- 因子分位组收益
- Top / Bottom 组合收益差
- RankIC / IC 均值
- 月度或阶段 IC 稳定性
- 时间切分下的表现对比
- 换手和可交易性粗检

第一版不需要特别复杂，但必须可复现。

## 14. 与主线策略的关系

因子研究要设计成以后可插入主线的形式，但当前不能直接接入。

未来可考虑的接入方式有三种：

1. 因子作为基线选股前的预过滤器
2. 因子作为候选股票排序器
3. 因子作为仓位权重修正器

第一版研究时，需要分析哪一种最适合，但不要现在就改主线代码。

## 15. 输出要求

研究结果必须方便主线决策，而不是只产出一堆 CSV。

至少要有：

- 一份研究总览 JSON 或 Markdown
- 一份每个因子的效果汇总
- 一份时间切分验证结论
- 一份“是否值得进入下一阶段”的结论

输出要能回答：

- 哪些因子值得保留
- 哪些因子无效
- 哪些因子只在样本内有效
- 哪些因子有潜力与基线结合

## 16. 建议的阶段拆分

### 阶段 1：框架搭建

- 盘点现有可复用模块
- 确定目录结构
- 建立最小因子面板和评估流程
- 跑通 1 到 2 个因子

### 阶段 2：基础验证

- 扩展到少量核心因子
- 跑分层收益和 RankIC
- 形成初步有效性判断

### 阶段 3：与基线比较

- 对照当前基线股票池和选股逻辑
- 研究因子作为过滤器 / 排序器的增益
- 形成“是否值得接入”的技术判断

### 阶段 4：接入设计

- 只出设计方案
- 暂不改主线交易实现

## 17. 阶段性完成标准

这条支线的阶段性完成标准不是“做出一个新策略”，而是：

- 建好了独立研究框架
- 结果可复现
- 目录清楚
- 输出清楚
- 能明确告诉主线：
  - 因子研究有没有价值
  - 哪条接入路径最合理
  - 下一步该做什么

## 18. 新对话开场要求

如果新对话直接接手，请先做两件事：

1. 盘点当前项目里所有能复用的“数据读取、股票池、基线回测、时间切分、输出目录”能力
2. 基于盘点结果，给出一版最小可行的因子研究实施方案，然后再开始写代码

不要一上来直接大改代码。
