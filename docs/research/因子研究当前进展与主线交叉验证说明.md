# 因子研究当前进展与主线交叉验证说明

这份文档用于回答两件事：

1. 当前研究支线到底做到了哪一步
2. 回到主线策略时，应该如何做交叉验证

重要边界：

- 当前所有因子研究结果都还在研究支线
- 没有改默认交易入口
- 没有改同花顺半自动流程
- 没有把研究结果直接接入交易执行

## 二、当前研究侧收敛出来的方案

### 1. 单因子与白名单

经过单因子评估、方向锁定和诊断后，研究支线曾收敛出一组排序型白名单，核心集中在：

- `close_ma20_gap`
- `ret60`
- `amount_ma20`
- `ret20`

之后又通过多因子消融进一步压缩。

### 2. 多因子合成

做过三种合成对照：

- `equal_weight`
- `rank_mean`
- `zscore_mean`

结果显示：

- `rank_mean` 是首版最优方案

### 3. 消融结果

对 `close_ma20_gap + ret60 + amount_ma20 + ret20` 做消融后，研究支线结论是：

- `amount_ma20` 很关键，不宜删除
- `ret20` 在当前组合里更像噪音增强项

因此研究侧新基准收敛为：

- `close_ma20_gap + ret60 + amount_ma20`
- 合成方式：`rank_mean`

## 三、各阶段已经得到的中间结论

### 1. 候选池质量层

`因子白名单对照.py` 说明：

- 白名单排序版明显优于基线候选池
- 但双层过滤没有进一步放大优势

因此研究侧阶段性判断为：

- 排序优先
- 过滤因子先做约束，不宜过重

### 2. 多因子研究层

`多因子合成对照.py`、`多因子稳定性诊断.py`、`多因子基准对照.py` 说明：

- `3` 因子版优于旧 `4` 因子版
- `rank_mean` 比等权和 zscore 更稳
- `top12` 相比过度集中的 `top4` 更稳健

所以研究支线内部的新基准是：

- `close_ma20_gap + ret60 + amount_ma20`
- `rank_mean`
- `top12`

### 3. 准交易层

`多因子准交易回测.py` 做的是：

- 固定持有 `5` 天
- 非重叠换仓
- `top12`
- 比较旧 `4` 因子和新 `3` 因子

这一层结论是：

- 新 `3` 因子方案优于旧 `4` 因子方案

这说明研究排序不是纯纸面优势。

### 4. 主线收益对照 P1

`主线收益对照_P1.py` 做的是：

- 统一为简化框架
- 固定持有 `5` 天
- 非重叠调仓
- `Top12`
- 只比较候选逻辑

这一层的结论不是研究因子明显胜出，而是：

- 因子方案命中率更高
- 但收益和回撤并没有全面压倒主线候选逻辑

这一步说明：

- 研究排序优势不会自动等价于最终回测优势

### 5. 主线收益对照 P2

`主线收益对照_P2.py` 做的是：

- 保留主线回测框架
- 保留主线市场过滤、30 分钟择时、退出、成本
- 唯一变化是替换 `select_candidates`
- 比较：
  - `mainline_default`
  - `mainline_factor_candidate`

这一层的最新结论是：

- `mainline_factor_candidate` 明显优于 `mainline_default`

从最新输出看：

- 因子候选版本：
  - 总收益约 `+6.81%`
  - 年化约 `+5.75%`
  - 最大回撤约 `-3.68%`
  - 夏普约 `1.17`
- 主线默认版本：
  - 总收益约 `-14.24%`
  - 年化约 `-12.22%`
  - 最大回撤约 `-15.13%`
  - 夏普约 `-1.86`

## 四、为什么你会觉得“这和主线以前记得的不一样”

这个疑问非常合理，必须单独说明。

`P2` 虽然复用了主线回测框架，但你仍然应该怀疑它是否和你记忆中的主线结果完全一致。原因可能包括：

### 1. 当前数据目录可能不同

当前结果使用的是：

- `/mnt/d/量化/同花顺/pythonProject/data/stock_data`

如果你以前主线回测使用的是：

- 别的切分数据目录
- 别的时间快照
- 别的更新版本数据

那么结果会不同。

### 2. 当前主线参数可能与你记忆中的版本不同

当前读取到的主线参数来自：

- `股票策略回测.py`

其中包括：

- `TOP_N`
- `MAX_HOLDING_DAYS`
- `MIN_RET20`
- `MIN_CLOSE_MA20_GAP`
- `MIN_MA20_SLOPE_PCT`
- `ENABLE_INTRADAY_EXIT`
- `ATR_STOP_MULTIPLIER`

如果你记忆中的回测结果来自旧参数集，结果会明显不同。

### 3. `P2` 是研究支线脚本发起的主线回测

它虽然调用的是主线回测函数，但仍建议你单独回到主线文件本身重新跑一次，确认：

- 主线默认回测
- 当前数据目录
- 当前参数
- 当前导出结果

是否和 `P2` 中的 `mainline_default` 一致。

### 4. 交易次数明显不同

`P2` 结果里：

- `mainline_default` 的平仓交易数只有 `106`
- `mainline_factor_candidate` 的平仓交易数是 `169`

这说明因子候选源改变后，主线执行壳子的触发频率也变了。

因此你要特别注意：

- 是不是候选逻辑改变导致更多交易
- 还是主线默认候选逻辑在当前数据上本来就偏保守

## 五、当前最准确的总体结论

到目前为止，能比较稳地说的是：

1. 研究支线已经完成从单因子到多因子、再到主线对照的完整首轮闭环
2. 研究侧新候选逻辑已经收敛为：
   - `close_ma20_gap + ret60 + amount_ma20`
   - `rank_mean`
3. 在保留主线执行壳子的 `P2` 对照里，因子候选版本优于主线默认候选版本
4. 但在真正准备接入主线之前，仍然建议你回主线脚本做一次人工交叉验证

## 六、你现在回主线应当怎么验证

建议按这个顺序做：

### 第一步：直接跑主线默认回测

目标：

- 确认你当前主线真实结果

你需要核对：

- 总收益
- 年化
- 最大回撤
- 夏普
- 交易数
- 胜率

然后和 `P2` 里的 `mainline_default` 对比。

### 第二步：核对数据根目录

确认主线当前实际使用的数据目录是否是：

- `/mnt/d/量化/同花顺/pythonProject/data/stock_data`

如果不是，先不要直接拿 `P2` 结论替代你的主线结论。

### 第三步：核对参数

重点核对：

- `TOP_N`
- `MAX_HOLDING_DAYS`
- `ENABLE_INTRADAY_EXIT`
- 各类趋势和波动率阈值

### 第四步：核对是否存在你手工改过但当前文件里没有的逻辑

比如：

- 额外市场过滤
- 银行/红利等偏好
- 排除某些板块
- 特定冷却或仓位规则

如果这些逻辑不一致，那么 `P2` 和你记忆中的主线表现不一致就是正常的。

## 七、当前不建议直接做的事

- 不建议立刻把研究候选逻辑接入交易主线
- 不建议直接替换默认候选逻辑
- 不建议在未完成主线人工交叉验证前，直接宣布研究候选已经胜出

## 八、建议的下一步

你现在最适合做的动作就是：

1. 回到主线脚本本身重新跑一次默认回测
2. 把主线结果和 `P2` 结果做人工对照
3. 确认差异到底来自：
   - 数据
   - 参数
   - 代码版本
   - 候选逻辑

如果这一步核对后，`P2` 结论仍成立，那么下一步才值得讨论：

- 是否做“候选层灰度接入”
- 或者是否做“主线可切换候选源改造”
