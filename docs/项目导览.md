# 项目导览

## 项目目标

本项目的目标不是追求“立刻自动实盘”，而是建立一条可信的完整链路：

1. 数据下载与清洗可信
2. 回测逻辑时序严格正确
3. 基线策略可复现
4. 模拟盘执行可验证
5. 交易自动化边界清楚

## 当前边界

当前系统已经能做到：

- 用基线策略生成次日订单意图
- 用同花顺模拟盘做预览
- 读取模拟盘余额
- 结构化保存当日信号、快照、对账、日志

当前系统还不能稳定做到：

- 无人值守自动提交
- 稳定读取模拟盘持仓/委托/成交表
- 以程序自动对账完全代替人工确认

## 近期最重要的两个方向

### 1. 整理系统

- 统一入口
- 补齐 README
- 保留旧版但不再混入主线
- 明确主流程和历史参考的边界

### 2. 攻克交易端读取

- 针对同花顺模拟盘的持仓/成交表格做更底层的控件分析
- 判断是 `easytrader` 默认 grid 适配问题，还是查询页结构特殊

在这一步解决前，系统仍应保持“半自动”定位。
