# 主线对因子研究的阶段性结论

本文档用于给主线做一次收口：

- 哪些因子研究结论，主线已经独立核验过
- 哪些事情，继续交给因子研究支线推进
- 主线当前不再继续深入哪些研究细节

目标是把边界切清楚，避免主线和研究支线继续混线。

## 1. 当前主线已经确认的事项

### 1.1 `P2` 不是最终可信验证

主线已经确认：

- [主线收益对照_P2.py](/mnt/d/量化/同花顺/pythonProject/主线收益对照_P2.py)
  对照的是 [股票策略回测.py](/mnt/d/量化/同花顺/pythonProject/股票策略回测.py)
- 它不是当前项目已经收口的可信主线
- 因此 `P2` 只能证明：
  - 因子候选替换旧主线候选源，在旧壳子上有提升
- 不能直接证明：
  - 因子候选已经在当前可信基线主线上验证通过

这个问题已经完成对账，不再继续作为主线争议点。

### 1.2 `P3` 已经对齐可信基线壳子

主线已经确认：

- [主线收益对照_P3_基线版.py](/mnt/d/量化/同花顺/pythonProject/主线收益对照_P3_基线版.py)
  使用的是 [股票策略回测_基线版.py](/mnt/d/量化/同花顺/pythonProject/股票策略回测_基线版.py)
- `P3` 的补丁点打在 `baseline._BASE.select_candidates`
- 这与基线版内部调用候选入口的方式一致

因此，`P3` 已经进入“可信主线壳子下的候选替换验证”范畴。

### 1.3 主线已经独立复跑过 `P3`

主线已经独立执行：

- 复跑 [主线收益对照_P3_基线版.py](/mnt/d/量化/同花顺/pythonProject/主线收益对照_P3_基线版.py)
- 单独复跑 [股票策略回测_基线版.py](/mnt/d/量化/同花顺/pythonProject/股票策略回测_基线版.py) 默认回测

核验结果：

- `P3` 中的 `baseline_default`
- 与主线独立运行的基线版默认结果完全一致

这说明：

- `P3` 至少没有把默认基线组跑偏
- `P3` 的默认对照组可以视为可信

### 1.4 当前独立复跑结果支持因子候选优于默认候选

主线独立复跑得到的 `P3` 结果如下：

- `baseline_factor_candidate`
  - 收益约 `+19.26%`
  - 年化约 `+16.12%`
  - 最大回撤约 `-6.65%`
  - Sharpe 约 `1.72`
  - 平仓交易数 `92`
  - 胜率约 `55.43%`
- `baseline_default`
  - 收益约 `+16.50%`
  - 年化约 `+13.83%`
  - 最大回撤约 `-12.79%`
  - Sharpe 约 `0.82`
  - 平仓交易数 `60`
  - 胜率约 `36.67%`

主线的当前结论是：

- 因子候选版本已经通过“可信基线壳子下的首轮独立交叉验证”

## 2. 主线当前不再继续深入的研究问题

下面这些问题，主线暂时不再继续深挖，统一交给因子研究支线负责：

### 2.1 单因子筛选与方向锁定

例如：

- 哪些单因子进入白名单
- 为什么 `ret20` 被移除
- 为什么 `amount_ma20` 被保留
- `rank_mean / zscore_mean / equal_weight` 的细节比较

这些属于研究收敛过程，不再由主线主导。

### 2.2 多因子组合的进一步扩展

例如：

- 是否继续测试更多因子
- 是否调整 `top12`
- 是否继续做更多消融组合
- 是否引入新的风格因子、行业因子、风险因子

这些都继续交给因子研究支线。

### 2.3 研究支线内部报告体系

例如：

- 因子评估汇总
- 白名单对照
- 稳定性诊断
- 准交易回测
- 多因子报告体系如何继续演进

这些都不再占用主线精力。

## 3. 当前主线只保留三个判断

主线现在只保留下面三个工程判断：

### 判断 1

因子候选已经具备进入灰度接入讨论的资格。

### 判断 2

当前不应直接把因子候选设为默认候选源。

### 判断 3

如果后续要接入，应该采用：

- 候选源可切换
- 默认逻辑不变
- 保留回退路径
- 增加最小必要日志

## 4. 继续交给因子研究支线的事项

后续与因子研究有关的深入工作，统一交给支线推进，包括：

1. 因子组合进一步优化
2. 因子稳定性继续扩展验证
3. 风格暴露与行业偏移分析
4. 更长时间窗口下的稳健性复核
5. 因子候选与主线候选差异的解释性分析
6. 灰度候选源的命名、版本管理与研究说明

主线只接收它的阶段性结论，不再主动参与细节研究。

## 5. 主线下一步应该做什么

主线下一步不再做“因子研究”，而只做“工程接入准备”。

建议顺序如下：

1. 保持当前默认候选逻辑不变
2. 设计候选源开关
3. 支持：
   - `default`
   - `factor_candidate`
4. 给候选层增加最小日志
5. 先做回测灰度
6. 再决定是否进入模拟灰度

## 6. 当前最合适的分工

### 主线负责

- 交易闭环稳定
- 基线策略执行与验证
- 候选源灰度接入设计
- 配置开关、日志、回退路径

### 因子研究支线负责

- 因子研究框架
- 组合继续优化
- 稳健性与解释性分析
- 研究报告和阶段结论

## 7. 一句话结论

当前主线已经完成“因子候选是否值得进入灰度接入讨论”的独立验证。  
从现在开始，因子本身的继续研究交给支线；主线只处理未来是否、以及如何工程化接入。
